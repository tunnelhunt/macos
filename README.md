# Tunnelhunt macOS Client

Sleek, lightweight, and modern macOS menu bar extra client for managing SSH tunnels.

![Tunnelhunt App Icon](Resources/AppIcon.icns)

## Установка и запуск (без сертификата разработчика Apple)

Поскольку приложение собирается автоматически в GitHub Actions и не подписывается платным сертификатом Apple Developer, macOS Gatekeeper по умолчанию заблокирует его запуск. 

Вы можете легко запустить приложение с помощью одного из следующих способов:

### Способ 1: Через Finder (самый простой)
1. Скачайте `Tunnelhunt.dmg` или `Tunnelhunt.zip` со страницы релизов GitHub.
2. Перенесите `Tunnelhunt.app` в папку **Программы** (`/Applications`).
3. Зажмите клавишу **Control** (или нажмите правой кнопкой мыши) на иконке `Tunnelhunt.app` в Finder и выберите пункт **Открыть** (Open).
4. В появившемся предупреждающем окне нажмите кнопку **Открыть** (Open).
   *После этого macOS запомнит ваш выбор, и приложение будет открываться двойным кликом как обычно.*

### Способ 2: Снятие карантина через Терминал
Если macOS показывает ошибку «Приложение повреждено» или блокирует запуск:
1. Откройте приложение **Терминал**.
2. Выполните команду:
   ```bash
   xattr -cr /Applications/Tunnelhunt.app
   ```
   *Эта команда снимает флаг карантина (quarantine attribute), который macOS присваивает файлам, скачанным из интернета.*
3. Запустите приложение стандартным способом.

## Возможности

- **Menu Bar App (LSUIElement):** Приложение работает исключительно в панели меню и не занимает место в Dock.
- **Universal Binary:** Поддерживает как процессоры Intel (`x86_64`), так и Apple Silicon (`arm64`).
- **Сворачивание и управление:** Удобное добавление, изменение и копирование параметров туннелей в один клик.

## Локальная сборка и запуск

Для сборки приложения на локальном компьютере вам понадобится установленный Xcode или Swift Command Line Tools.

### Сборка и запуск в режиме отладки:
```bash
swift run
```

### Сборка релизной версии:
```bash
swift build -c release
```

Бинарный файл будет находиться по пути `.build/release/TunnelhuntTray`.

---

## Автоматическая сборка в CI/CD

Проект настроен на сборку в GitHub Actions с помощью файла конфигурации `.github/workflows/release-macos.yml`. 

- **Триггеры сборки:** Запускается автоматически при публикации нового тега (например, `v1.0.0`) или вручную через вкладку Action на GitHub (`workflow_dispatch`).
- **Результаты:** В релизы автоматически выгружаются `.dmg` установщик и `.zip` архив с универсальным приложением.
